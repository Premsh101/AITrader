AITrader – User Manual

How to use the application, with screenshots — plus the one-time setup and simple answers to common questions.

1. Is my system ready?

✅ The software — ready. All known bugs are fixed and tested. The dashboard, the safety checks and the trading engine all work.
❌ The AI models — one-time training needed. The three "brains" must be trained once on Kaggle (Part 3 below, mostly copy-paste). Until then the app runs and shows the dashboard, but refuses to trade — on purpose, for your safety.
⚠️ Your settings — a few minutes of work. Set a password key and a couple of values in the .env file before deploying (Part 3, step D).

Part 1 – Using the dashboard (day-to-day)

1.1 Open the dashboard

Open your dashboard address in any browser (for example http://your-server:8006 or the domain your proxy gives it). There is no login page — but every action (like switching modes) is protected by your secret API key, so visitors can only look, never touch. The page refreshes itself every 30 seconds; the Refresh button forces an update immediately.

1.2 The top bar — know your mode at a glance

  • Green “PAPER MODE” — the system simulates trades with real market prices but no real money. This is the default and where you should stay for the first weeks.
  • Red “LIVE TRADING” — real orders, real money, through your Shoonya broker account.

1.3 The four big numbers

CardWhat it means
Daily P&LMoney made or lost today from positions that were closed today. Green = profit, red = loss.
Total ProfitAll-time result of every closed trade added together.
Win RateOut of all closed trades, how many ended in profit. Above 50% is good, but a few big losses can still outweigh many small wins — always read it together with Total Profit.
Open PositionsHow many stocks the system is holding right now. The maximum is 5.

1.4 Open positions — what the system is holding now

These are unfinished trades. The Guardian AI reviews each of them every cycle and decides when to sell. You don't need to do anything here — it's information, not a control.

1.5 The Live Activity Log — the system talking to you

Three AI "employees" work for you, and each one signs its messages:

NameJobTypical message
HunterScans ~145 NSE stocks and shortlists ones worth buying.“Found 4 buy signal(s): AXISBANK, …”
ExecutivePicks the best of the Hunter's list, but only if a free slot exists and it's confident enough.“Placing Paper BUY order → AXISBANK …”
GuardianWatches every open position and decides when to sell.“Holding position → LT” / “Closing position → VOLTAS”
SystemHousekeeping and warnings.“Trading cycle complete”

The system only works during Indian market hours (Monday–Friday, 09:15–15:30 IST). Outside those hours the log stays quiet — that is normal, not a problem.

1.6 Trade History — your record book

  • BUY ₹ / SELL ₹ — the recorded buy and sell prices.
  • P&L — profit or loss in rupees after the trade closed (green positive, red negative).
  • STATUS — Open (still holding) or Closed (finished).
  • MODE — whether it was a Paper (practice) or Live (real) trade. Practice and real results are never mixed up.

1.7 Your two-minute daily routine

  1. Open the dashboard once a day (after 15:30 IST is ideal — the day is complete).
  2. Glance at Daily P&L and Total Profit.
  3. Skim the Activity Log — you should see normal cycle messages. Anything that looks like an error → check Part 5.
  4. Once a week, look at the Trade History and ask: would I be happy if this had been real money? Only when the answer has been “yes” for several weeks should you think about Live mode.

Part 2 – Switching between Paper and Live

  1. Click the Paper / Live switch in the top-right corner.
  2. When switching to Live, your browser shows a confirmation popup: “Switch to LIVE mode? The system will place REAL orders with REAL money.” Click OK only if you truly mean it. (The server double-checks this too — a request without the confirmation is refused.)
  3. The pill in the header turns red. From the next trading cycle, orders go to your Shoonya account.
  4. To stop: flip the switch back to Paper at any moment. Positions already open at the broker are intraday and are squared off by the broker around 15:20 IST in any case.
Before your first Live switch, make sure: the models passed the evaluation (Part 3), paper results looked good for weeks, your Shoonya credentials are in .env, and TRADE_CAPITAL_PER_SLOT is an amount you can afford to lose — the system uses roughly that many rupees per position, up to 5 positions.

Part 3 – One-time setup (do this first)

You do this once. Afterwards, the application just runs.

A. Merge the pull request

Merge PR #12 on GitHub into main.

B. Train the three AI models on Kaggle (mostly copy-paste)

The training program is already written — running it creates and trains the models for you. On kaggle.com: create a new notebook, switch Internet ON in the settings, then run these cells:

# Cell 1 – get the code and libraries
!git clone https://github.com/Premsh101/AITrader.git
%cd AITrader
!pip install -q "stable-baselines3==2.8.0" "numpy>=2.0" "pandas_ta==0.4.67b0" "yfinance>=0.2.40"

# Cell 2 – quick 2-minute test that everything works
!python training/train_triad.py --smoke

# Cell 3 – the real training (takes a few hours, saves progress every 100k steps)
!python training/train_triad.py --timesteps 1000000

# Cell 4 – the report card
!python training/evaluate_triad.py

C. Read the verdict — the go / no-go rule

  • VERDICT: PASS → download the three files from AITrader/models/ in Kaggle's file panel (hunter_apex_1500_brain.zip, guardian_apex_1500_brain.zip, executive_apex_manager.zip) and commit them to the repo's models/ folder with the same names.
  • VERDICT: FAIL → the models did not beat simply holding the NIFTY index on data they had never seen. Do not deploy them. Adjust and retrain (see training/README.md).

D. Fill in your settings on the server

cp .env.example .env
openssl rand -hex 32        # generate your secret key

Edit .env: set AITRADER_API_KEY and NEXT_PUBLIC_API_KEY to that same key, set ALLOWED_ORIGINS to your dashboard address, and (only for Live mode) your SHOONYA_* broker credentials.

E. Start it

docker compose up --build -d

Database tables are created automatically. Note: the app now listens only on the server itself (127.0.0.1) — your reverse proxy (Coolify) must forward public traffic to ports 8005 (API) and 8006 (dashboard).

F. Check it's alive

curl http://127.0.0.1:8005/health
# → {"status":"ok","brains_ready":true}

If brains_ready is false, the model files are missing or wrong — see Part 5.

Part 4 – Simple answers to common questions

Do I need to create the models myself and then train them again?
No creating, yes one training run. The repo already contains the “model factory” (training/train_triad.py). Running it on Kaggle builds and trains all three models in one go and saves the three files. You copy those files into the repo once, and you're done — no regular retraining is needed. (Retraining every few months to refresh the models with new market data is a good habit, but optional.)
Why can't we use the old models we already had?
Think of a plug and a socket. The app describes each stock with 15 numbers; the old models were built to accept only 5 numbers (and the third one, 3). The plug never fit the socket — every prediction failed silently, which is exactly why the old system never traded. The “blueprints” (training notebooks) that made those models are lost, so they can't be adapted either. They were moved to models/legacy/ as a record. The new training scripts use the app's own 15-number code directly, so this mismatch can never happen again — and the app now checks the fit on startup and refuses to run with a wrong model.
Is Paper mode really risk-free?
Yes. In Paper mode the broker is never contacted — this is enforced in code and covered by tests. Prices come from Yahoo Finance and trades exist only in your own database.
How much money does Live mode use?
About TRADE_CAPITAL_PER_SLOT rupees per position (default ₹10,000), and at most 5 positions at once — so roughly ₹50,000 exposure at the default settings. Change the value in .env.
Why is nothing happening right now?
Most likely the market is closed. The system only acts Monday–Friday, 09:15–15:30 IST. Weekends and evenings are silent by design.
What does the evaluation “PASS / FAIL” actually compare?
It replays the complete strategy on recent data the models never saw during training, subtracts realistic trading costs, and compares the result with just buying and holding the NIFTY index. If the strategy can't beat “do nothing”, it isn't worth real money — so FAIL means don't deploy.

Part 5 – Troubleshooting

The one error you will most likely see

Fix: complete Part 3 steps B–C (train on Kaggle, commit the three zips into models/ with the exact filenames), then redeploy.

Other issues

SymptomCauseFix
Dashboard buttons do nothing (401 error)The two API keys don't match.Make AITRADER_API_KEY = NEXT_PUBLIC_API_KEY in .env, then docker compose up --build -d (the dashboard bakes the key in when it builds).
Action returns 503 “API key not configured”AITRADER_API_KEY is empty on the server.Set it in .env and restart.
Switching to Live is refusedThe extra safety check needs an explicit confirmation.Use the dashboard switch and accept the popup.
“No market data returned – skipping cycle”Yahoo Finance temporarily unreachable or rate-limiting.Usually clears on its own; if persistent, increase FETCH_TTL_SECONDS in .env.
One stock always shows warningsIts ticker was renamed or delisted on Yahoo.Update or remove it in app/stock_list.py.
Numbers differ slightly from broker statement (Live)Fill prices are recorded from the quote just after the order — a close approximation, not the exact fill.Expected behaviour; treat the broker statement as the source of truth for accounting.
